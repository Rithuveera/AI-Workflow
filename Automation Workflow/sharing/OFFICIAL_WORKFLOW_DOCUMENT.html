1. What is n8n?

n8n is the "brain" of our operation. It's a visual workflow tool that allows us to connect complex tools like Playwright with communication platforms like Slack without writing massive amounts of glue code.

Visual Interface

Unlike scripts hidden in terminal windows, n8n shows you exactly where the data is moving through nodes.

Unified Control

It acts as the single entry point for all our automated tests, whether scheduled or manual.

5. Running Multiple Scripts

You have two powerful ways to run multiple test scripts in this workflow:

Option A: Bulk Run

In the "Set Script to Run" node, enter multiple paths separated by spaces or use a wildcard:

tests/*.spec.js

Best for: Fast execution, single report.

Option B: Sequential Loop

Import n8n-multi-script-loop.json. This uses an array of scripts and runs them one by one.

[ { script: "A.js" }, { script: "B.js" } ]

Best for: Individual Slack reports for each test.

6. Product-Wise Execution

Our test suite is organized by product folders within the tests/ directory (e.g., tests/ERP/).

How to run a Product Suite:

  1. Import n8n-product-wise-runner.json.
  2. Open the "Select Product" node.
  3. Update the ProductName value (e.g., ERP, CRM).
  4. Execute the workflow. Playwright will automatically find and run all .spec.js files inside that folder.
